# Content Schedule — foodcosting.app

> Last updated: 2026-04-18
> Cadence: 2 blog posts/week, 1 social post/day
> Review required: All social posts need Ahmed's sign-off before scheduling

---

## Weekly Production Rhythm

### Monday — Blog Post 1
1. **topic-researcher** runs (GSC analysis → brief)
2. **blog-writer** runs (brief → draft post)
3. Draft saved to `pipeline/posts/` → pushed to Notion Blog DB as "Review"
4. Ahmed reviews & approves for publishing

### Tuesday — Social Batch 1
1. **short-form-writer** runs on Monday's blog post (→ 2 video script variants)
2. **remotion-renderer** renders recommended variant (→ MP4)
3. **carousel skill** creates carousel from Monday's blog post (→ PNGs)
4. Both assets staged in `pipeline/out/` for Ahmed's review

### Wednesday — Backlog Repurpose Day
1. Pick next unpublished-social blog post from Notion backlog
2. Run **short-form-writer** OR **carousel skill** on backlog post
3. Stage for Ahmed's review

### Thursday — Blog Post 2
1. **topic-researcher** runs (GSC analysis → brief)
2. **blog-writer** runs (brief → draft post)
3. Draft saved to `pipeline/posts/` → pushed to Notion Blog DB as "Review"
4. Ahmed reviews & approves for publishing

### Friday — Social Batch 2
1. **short-form-writer** runs on Thursday's blog post (→ 2 video script variants)
2. **remotion-renderer** renders recommended variant (→ MP4)
3. **carousel skill** creates carousel from Thursday's blog post (→ PNGs)
4. Both assets staged in `pipeline/out/` for Ahmed's review

### Saturday — Backlog Repurpose Day
1. Pick next unpublished-social blog post from Notion backlog
2. Run **carousel skill** on backlog post
3. Stage for Ahmed's review

### Sunday — Backlog Repurpose Day
1. Pick next unpublished-social blog post from Notion backlog
2. Run **short-form-writer** → **remotion-renderer** on backlog post
3. Stage for Ahmed's review
